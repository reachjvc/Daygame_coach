# Goal Intelligence — Vision (concept, not a build plan)

**What we're building:** a system that takes a person's raw words and works out the *real desire
underneath*, instead of stuffing them into the first plausible template. Goals become **clues to a
person**, not orders to obey. The output is a plan that actually *fits* — and sticks.

**Why:** a stated goal ("a successful business") is almost always a *means* to a deeper *end*
(freedom, worth, security). Match on the surface and you build the wrong plan confidently. The
value is in the *why*, the *trade-offs*, and *what they're not saying* — none of which keyword
matching can reach.

> ⚠️ **Living model.** Any structure below is provisional and will miss real-world complexity.
> Treat its categories as a first draft to be extended, not a finished ontology. The system must
> be able to say "I don't have a box for this yet" rather than force-fit.

---

## What we need to build (rough order)

> Start at the **bottom layer: the reference lists everything matches against.** Without these the
> engines have nothing to reason over. The first lift is figuring out *"bad" goals → real needs*,
> and that is mostly **authored/curated data + an evidence base**, not code. This is the "long list
> to match things" — really several linked lists.

```
PHASE 0 · THE LISTS TO MATCH AGAINST   (the backbone — start here; mostly content work)
  [ ] NEEDS taxonomy       the ~12–15 root needs every goal ladders to
                           (autonomy · competence · relatedness · security · status · meaning ·
                            vitality · freedom · belonging · growth · safety · …)   ← the targets
  [ ] GOAL→NEED seed map   common stated goals/phrases → the need(s) they usually serve
                           ("a business"→autonomy|status|security · "girlfriend"→relatedness)
  [ ] NEED↔AVENUE efficacy per need: which avenues actually satisfy it (STRONG) vs don't (WEAK)
                           — THIS is the evidence base behind every "bad goal / trap" call
  [ ] FEELING lexicon      affective-state phrases that are SYMPTOMS, not goals
                           ("feel good"·"empty"·"lost"·"feel like shit") → trip the diagnostic
  [ ] ATOM tags + cues     phrase patterns → outcome | feeling | identity | value | avoidance
  [ ] VALUES↔NEEDS map     link the 301 Inner-Game values to the needs taxonomy
  [ ] CONFLICT pairs       which needs/goals compete for the same hours / energy / identity
  [ ] READINESS cues       phrases signalling denial · ambivalence · ready-to-act

PHASE 1 · THE ENGINES   (read the lists → fill the person-model)
  [ ] DECOMPOSE · WHY-LADDER · NEED-MAP · EFFICACY · VALUES · CONFLICT · PRIORITY · READINESS
  [ ] QUESTION BANK        per person-model blank: the smart question(s) that fill it

PHASE 2 · ORCHESTRATION + UX   (turn lists+engines into a real conversation)
  [ ] PERSON-MODEL store   where the blanks live + how they update across sessions
  [ ] CONVERSATION flow    which engine speaks when (turn-by-turn, not all at once)
  [ ] PLAN generator       person-model → falsifiable plan ("does this feel yours?")
  [ ] INNER-GAME wiring    read core values in · feed goal-evidence back
```

**The crux of "bad goals" lives in the first three Phase-0 lists:** a goal is "bad" when its
*avenue* is WEAK for the *need* it's really serving (NEED↔AVENUE), and we only know the need by
laddering the stated goal (GOAL→NEED) onto the NEEDS taxonomy. Build those three first; everything
else reads them.

## The shift

```
  OLD   sentence ──▶ keyword match ──▶ template                 (fast, confident, often wrong)

  NEW   sentence ──▶ ┌─────────────────────┐ ──▶ PERSON-MODEL ──▶ plan that fits
                     │ diagnostic engines  │         (the blanks)
                     └─────────▲───────────┘            │
                               └── ask smart questions ◀┘   (each Q fills a blank)
```

The plan is the **last mile**, generated from a filled-in model — not the first step.

## The PERSON-MODEL (the "blanks" the system carries about you)

```
┌───────────────────────────── PERSON ─────────────────────────────┐
│ SAYS they want      │ the literal input, split into goal-atoms     │
│ REAL desire (why)   │ laddered root of each atom                   │
│ UNMET needs         │ autonomy · competence · relatedness · ...    │
│ DRAINS (subtract)   │ what's currently costing them (leaks/vices)  │
│ FEELING vs REALITY  │ is a stated "feeling" a symptom, not a goal? │
│ CAPACITY            │ real time / energy / money available         │
│ IDENTITY            │ who they're trying to become                 │
│ VALUES / fit        │ which core values it expresses (↔ Inner Game)│
│ TENSIONS            │ which goals fight each other                 │
│ PRIORITY            │ the ONE for this season                      │
│ READINESS           │ acting? ambivalent? in denial?               │
└──────────────────────────────────────────────────────────────────┘
```

Each engine below is just *something that fills one row by asking a good question.*

## The engines (many fire from one sentence)

```
   sentence
      │
      ├─▶ DECOMPOSE     split into atoms; tag each: outcome│feeling│identity│value│avoidance
      ├─▶ WHY-LADDER    climb means→ends until a root NEED  (see ladder)
      ├─▶ NEED MAP      which needs are unmet (the ache) vs thriving
      ├─▶ EFFICACY      is the chosen avenue evidence-based for that need? (trap check)
      ├─▶ ELIMINATION   what's DRAINING them — sometimes removing > adding
      ├─▶ CONFLICT      do these goals fight for the same hours/energy/identity?
      ├─▶ PRIORITY      given conflict + capacity, which ONE now?
      ├─▶ IDENTITY      "who are you becoming?" — the frame that makes it stick
      ├─▶ VALUES        does it express a CORE value? (↔ Inner Game) → fuel / tension / orphan
      └─▶ READINESS     ready to act, or still ambivalent → plan vs motivate
                                  │
                                  ▼
                            PERSON-MODEL  ──▶  plan (proposed, falsifiable: "does this feel yours?")
```

## The desire ladder (finding the real why)

```
  TARGET   "bench 140kg"            ← what they typed
    ▲
  GOAL     "get jacked"
    ▲
  WHY      "be respected / wanted"
    ▲
  NEED     relatedness · status · self-worth   ← the actual driver, where the plan should aim
```

## The trap check (a "bad goal" = a weak avenue to a real need)

```
  need ──picks──▶ avenue ──efficacy?──▶ ✓ strong  → back it, plan it
                                   └──▶ ✗ weak    → don't preach — ASK, then surface better routes

  e.g.  "feel worthy" ─▶ "earn more money" ─▶ weak (threshold returns) → probe → offer mastery/connection
        "stop feeling lonely" ─▶ "more dating volume" ─▶ weak → probe → offer depth/friendship
```

The happiness-trap is just *one* instance of this. So is the rebound, the revenge-body, the
hustle-as-escape. **Perceived** progress toward a feeling, via an avenue that won't deliver it.

## Values & alignment (↔ Inner Game)

Values are the **bridge between the hidden NEED and the chosen GOAL**: a need is *expressed
through* the values a man holds, and a goal only sticks when it honours them. This is also the
seam where this system connects to the rest of the site — the **Inner Game** slice already builds
a man's values (301 values → cut to a CORE set via picking, peak-experience, shadow & hurdles
work, pairwise cutting, and LLM inference from his own words). Goal Intelligence should **read**
that core set and **feed evidence back**.

```
  need ──expressed-through──▶ VALUES ──shape──▶ goal

  goal ──expresses?──▶ a core value     → ✓ congruent  → strong, durable fuel
                  ├──▶ violates a value  → ⚠ tension    → name it, let him choose
                  └──▶ touches none      → ∅ orphan     → "is this yours, or a 'should'?"
```

```
  INNER GAME (values)                          GOAL INTELLIGENCE
  ┌─────────────────────────┐   core values    ┌─────────────────────────┐
  │ 301 values → CORE set   │ ───────────────▶ │ align  goal ⟷ values     │
  │ pick·peak·shadow·       │                  │ congruent / tension /    │
  │ hurdles·pairwise·LLM    │ ◀─────────────── │ orphan  → ask            │
  └─────────────────────────┘   new evidence   └─────────────────────────┘
     (his chosen goals are themselves evidence of what he values — and a goal
      that fights a stated value is a flag worth raising, not a thing to bury)
```

Values also **disambiguate the avenue**: two routes to the same need — pick the one that fits him.
Make money by *craft* (values: mastery, quality) vs *hustle* (values: speed, autonomy). Same need,
different value-fit, different plan.

## What makes a goal stick — held through several lenses (no single one is right)

There is no one theory of why goals stick; each lens below is a different camera on the same man.
A goal is **strong when several lenses agree**, **fragile when it fails many** — and where lenses
*disagree* is itself signal (e.g. specific + motivating but values-violating = a fast burnout).
The system's job is **not to pick a lens** but to run a goal past several and report the
agreement/conflict. Hold them all loosely; none is the doctrine. (Add lenses as we learn — this
list is provisional.)

```
  LENS                 sticks when…                          fails as…
  ──────────────────────────────────────────────────────────────────────────────────
  NEED (SDT)           meets autonomy/competence/relatedness  extrinsic, hollow
  VALUES-CONGRUENCE    expresses a core value (↔ Inner Game)  a "should", or value-violating
  IDENTITY             flows from "who I'm becoming"          outcome with no self behind it
  SPECIFICITY (Locke)  concrete, hard-but-doable, w/ feedback "do my best", vague
  DIRECTION            framed TOWARD something                away-from → rebound / burnout
  COHERENCE            doesn't fight his other goals/roles    silent conflict, self-sabotage
  MEANING              a why deep enough to survive friction  thin → quits at first obstacle
  READINESS (stages)   matched to where he actually is        action pushed onto ambivalence
  CONTROLLABILITY      built on actions he commands           hostage to outcomes he can't
  ADAPTATION           renews / has a eudaimonic anchor       pure feeling-chase → treadmill
```

Almost none of these are about *which template*. They're about the **fit between person and goal**
— and several of them (need, values, meaning, identity) are exactly what the engines above try to
fill in before any plan is drawn.

## 10 inputs — how real men actually type (unprompted, 22–40)

> Verbatim-style, with natural mess. Each shows: surface atoms → desire underneath → what fires.

```
#1 · PRIMARY · broad / four-at-once
"i want to feel good, wake up loving my life, a succesful business and a loving relationship"
  atoms: feeling ×2 · achievement · relationship
  underneath: ❓ (business=freedom or worth?) · relationship likely the need-dense one
  fires: FEELING→diagnose · WHY-LADDER(business) · CONFLICT(4 goals, same hours) · PRIORITY · ELIMINATION

#2 · 24 · terse, aesthetic, no why
"get jacked. bench 140. abs by summer."
  atoms: outcome ×3 (specific, approach, zero why)
  underneath: status / desirability / self-image
  fires: WHY-LADDER(missing) · IDENTITY("for whom?") · EFFICACY(body→worth?)

#3 · 29 · heartbroken, avoidance, ambivalent
"my ex left me a few months ago and i feel like shit. i just want to stop feeling this way, maybe meet someone, idk"
  atoms: feeling(pain) · avoidance · vague relationship · low confidence
  underneath: grief · self-worth · relatedness
  fires: READINESS(low) · FEELING→diagnose · EFFICACY(dating to fix grief = trap) · ELIMINATION

#4 · 26 · hustle, grandiose, extrinsic
"I want to make 30k a month, quit my 9-5, be my own boss and travel. financial freedom is everything."
  atoms: outcome(money) · escape · autonomy · "money = everything"
  underneath: autonomy vs escape vs status — unresolved
  values: claims FREEDOM — yet a 30k/mo grind may VIOLATE it → tension to surface
  fires: WHY-LADDER · EFFICACY(money→freedom assumption) · CAPACITY · VALUES(conflict)

#5 · 31 · addiction, repeated failure, stakes
"i need to quit drinking and porn before it ruins my marriage. been trying for years."
  atoms: avoidance ×2 (vices) · relationship stake · history of failure
  underneath: self-respect · the marriage · agency
  fires: READINESS(action but shame) · APPROACH-REFRAME(toward, not away) · ELIMINATION · IDENTITY

#6 · 34 · has it all, feels empty (the sprung trap)
"i make good money, have a girlfriend, gym 5x a week… but i feel empty and don't know the point anymore"
  atoms: achievements listed · meaning-void
  underneath: eudaimonic gap — avenues achieved, feeling never arrived
  values: ORPHAN goals — chased avenues that may never have matched his core values
  fires: FEELING→diagnose · MEANING/PURPOSE · VALUES(orphan) · EFFICACY(trap already sprung) · DON'T pile on

#7 · 22 · social anxiety, lonely, identity
"i want to be more confident and less awkward around people, especially girls. i don't really have friends"
  atoms: identity/inner-game · social · loneliness
  underneath: relatedness + competence + belonging
  fires: NEED MAP(belonging) · WHY-LADDER(confidence for what?) · ELIMINATION(isolation)

#8 · 38 · new dad, for-others, practical
"want to get healthy for my kids, lose the dad bod, more energy, and stop being so stressed and short-tempered"
  atoms: health(for others) · aesthetic · energy · emotion-regulation
  underneath: be a good father (relatedness/identity) · vitality
  fires: IDENTITY(father) · CONFLICT(time w/ kids vs gym) · ELIMINATION(stress source) · APPROACH-REFRAME

#9 · 33 · cynical, low buy-in, vague
"honestly idk why i'm here. self improvement is mostly bs but my life's a mess and something's gotta change"
  atoms: meta-skepticism · vague pain · low readiness
  underneath: unknown — needs ONE concrete pain surfaced
  fires: READINESS(contemplation) · MEET-RESISTANCE · ELIMINATION(find the one real ache first)

#10 · 40 · rock-bottom, overwhelmed, urgent
"doctor says my blood pressure is dangerous, i'm overweight, i drink too much and my marriage is rough. need to turn it all around"
  atoms: health-scare · weight · vice · relationship — all at once
  underneath: survival + dignity; everything correlated
  fires: PRIORITY/TRIAGE(don't pile on) · CONFLICT(overwhelm) · ELIMINATION-heavy · READINESS(high, urgent)
```

**Pattern across the ten:** the same machinery must handle a one-word lifter and a rambling
rock-bottom; approach goals and avoidance goals; people who've achieved the avenue and still feel
nothing; and people who don't know what they want at all. *Which template* is the easy 10%. The
other 90% is reading the person.

## Non-goals / open questions
- Not therapy, not diagnosis — it *asks*, it doesn't *tell* or label.
- How much to infer silently vs confirm out loud? (silent-confident-wrong is the current sin)
- Where does the model persist, and how does it update over time / across sessions?
- The efficacy/"trap" map needs a defensible evidence base — and a light touch (ask, never preach).
- Extensibility: how does the system flag "this person doesn't fit my boxes"?
- Values ↔ goals: read-only from Inner Game, or does goal evidence write *back* (and can that be wrong)?
- What if a man has no core values set yet — does a goal seed a lightweight values pass, or wait?
